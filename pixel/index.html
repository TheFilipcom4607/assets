How reliable is this? Email open tracking is a soft signal, not a guarantee — the same caveats apply to every big email tool.
• Apple Mail Privacy Protection pre-loads images the moment mail arrives, so those show as "opened" even if unread, from an Apple relay location.
• Gmail & most webmail fetch images through a proxy, so the IP/location you see is the mail provider's, not the reader's, and caching means you'll usually catch the first open more reliably than repeats.
• Image blocking means some real opens are never recorded.
Please use responsibly. Tracking whether someone opened an email may require disclosure or consent in some places (e.g. the EU/UK). Track your own correspondence, not people who'd object.

How do I deploy the backend? (3 minutes, free)
  1. Create a free Cloudflare account (no card needed).
  2. Workers & Pages → Create → Worker. Name it e.g. pixel-track, Deploy.
  3. Edit code, delete the sample, paste in pixel/backend/worker.js, Save & Deploy.
  4. Create a KV namespace: Storage & Databases → KV → Create, call it pixel_data.
  5. Back in the Worker: Settings → Bindings → Add → KV namespace. Variable name PIXELS, pick pixel_data.
  6. Settings → Variables → Add DASH_TOKEN = a long random password.
  7. Copy your Worker URL (https://pixel-track.you.workers.dev) and the token into the fields above.
